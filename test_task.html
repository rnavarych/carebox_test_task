AI Automation QA Engineer Home Assignment

Overview

This assignment evaluates your ability to create email templates, design automated testing strategies, and produce meaningful comparison reports. You will build a simple email template system with variations and develop a testing approach to validate and compare template outputs.

Key Concept: AI-Driven QA Automation

Important: The core objective of this assignment is to design a testing system that can be fully executed by Claude Code (LLM), eliminating the need for human QA intervention.

The vision is to create an automated QA pipeline where:

  • Claude Code can independently run all validation and comparison tests
  • The LLM analyzes rendered outputs and identifies issues autonomously
  • Reports are generated without human review of individual templates
  • The entire QA cycle\'97from test execution to reporting\'97requires zero human involvement

Your testing strategy should leverage Claude Code's capabilities to reason about templates, compare outputs, and make intelligent assessments that would traditionally require a human QA engineer.

Assignment Tasks

Part 1: Create Email Templates

Create a simple email template called "Site Visitor Welcome" - a notification email sent when someone visits a website.

You need to create 3 variations of this template:

Template

Folder

Description

Base Template

site_visitor_welcome/

Standard welcome email with default styling. Should include: header, greeting, main message, call-to-action button, footer

Partner A Variation

site_visitor_welcome_partner_a/

Same structure as base template. Different color scheme (e.g., different primary color, button color, link color). Same content as base template

Partner B Variation

site_visitor_welcome_partner_b/

Same structure and colors as base template. Different content (e.g., different greeting text, different CTA text, additional section)

Template Requirements

  • Use MJML format with EJS templating
  • Templates should be responsive (mobile-friendly)
  • Include at minimum:
    • Logo/header section
    • Greeting with personalization (e.g., <%= context.visitorName %>)
    • Welcome message body
    • Call-to-action button
    • Footer with basic links


Part 2: Automated Testing Strategy

Design and implement an automated testing approach that can:

  1. Render Validation
    • Verify that templates compile/render successfully
    • Ensure no syntax errors or missing variables
    • Validate that output is valid HTML
  2. Visual/Structural Comparison
    • Compare the rendered output between template variations
    • Detect differences in styling, content, and structure
    • Identify what specifically changed between versions

Think creatively about how you would automate this testing. Consider:

  • What tools or frameworks could help?
  • How would you capture and compare template outputs?
  • How can you make the comparison meaningful and actionable?
  • How can you combine multiple tools together? (e.g., Playwright, Claude AI, image processing libraries, diff tools, etc.)

Critical Requirement: Claude Code as the QA Engine

Your testing implementation must be designed so that Claude Code (LLM) can execute the entire QA process autonomously. This means:

  1. No Human-in-the-Loop - The tests should run end-to-end without requiring human judgment or manual verification
  2. LLM-Executable Scripts - All test scripts should be executable by Claude Code via command-line tools
  3. AI-Powered Analysis - Leverage Claude Code's reasoning capabilities to analyze outputs, identify discrepancies, and assess quality
  4. Automated Decision Making - The system should determine pass/fail criteria and generate actionable insights without human review

The goal is to completely replace the traditional human QA role with an AI-driven pipeline. Your solution should demonstrate how Claude Code can serve as an intelligent QA engineer that understands context, catches issues, and provides meaningful feedback\'97all without human oversight.

Note: You have freedom in choosing your testing approach. The goal is to demonstrate your ability to think critically about QA automation and orchestrate different tools to work together in a smart way that enables fully autonomous LLM-driven testing.


Part 3: Difference Report

Generate a comparison report that documents:

  1. Summary
    • Overview of templates tested
    • High-level findings
  2. Detailed Differences
    • What changed between Base and Partner A (styling differences)
    • What changed between Base and Partner B (content differences)
    • Specific elements that differ (colors, text, sections)
  3. Recommendations
    • If any unintended differences were found, what should be fixed?
    • Are there consistency issues that need attention?

The report should be clear enough that a developer could use it to understand exactly what differs and what action to take.
